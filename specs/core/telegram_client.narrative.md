# Telegram Client

## Intent

Код для работы с Telegram. Получение списка чатов, поиск по ключевым словам и регуляркам.

Люди не всегда делают "ответ" — могут просто писать в чат. Для поиска переписки нужно ориентироваться на промежуток времени. Минимизируем чтение ИИ: выгрузка по часам, чтение только нужных блоков.

Workflow ИИ:

1. Поиск по ключевым словам → ссылки на текстовые файлы (в `data/`)
2. Читает блок (час) с самым свежим совпадением
3. Может запросить ±N сообщений относительно блока/сообщения

Лимиты на количество сообщений — защита от переполнения контекста. Позже можно добавить графовую базу + семантический поиск.

### Уточнения по lazy load и инкременту

Q12: Предлагаю такую логику:

### Ещё один момент

Q18: Telethon работает асинхронно. Весь проект делаем на `async/await`?

A18: Видимо да. А есть другие варианты?

> Альтернатива — синхронные обёртки через `loop.run_until_complete()`, но это антипаттерн для I/O-bound задач. Async правильнее: Telegram API, файловые операции, будущий MCP-сервер — всё I/O. Делаем async.
